## 🏗️ 架構設計

### MVVM 架構

```
ApiTestFragment (View)
        ↕ (DataBinding)
ApiTestViewModel (ViewModel)
        ↕ (LiveData/SingleLiveEvent)
        ↓
    Business Logic
```

### 觀察者模式

```java
// ViewModel 中定義
public MutableLiveData<String> logMessages;
public SingleLiveEvent<String> toastEvent;

// Fragment 中觀察
viewModel.logMessages.observe(..., messages -> {
    binding.tvLogMessages.setText(messages);
    scrollToBottom();
});
```

### 日誌管理

```java
// ViewModel 提供多種訊息方法
viewModel.addLogMessage(String message);      // 一般訊息
viewModel.addErrorMessage(String message);    // ❌ 錯誤
viewModel.addSuccessMessage(String message);  // ✅ 成功
viewModel.addWarningMessage(String message);  // ⚠️ 警告
viewModel.addInfoMessage(String message);     // ℹ️ 資訊

// 自動添加時間戳
[12:34:56.789] ℹ️ [資訊] 訊息內容
```

```java
public class ApiTestFragment extends

    }

```

### 2. DataBinding 使用

```xml
<layout>
    <data>
        <variable name="viewModel" 
                  type="com.cyberpower.functiontest.ApiTestViewModel" />
    </data>
    <!-- UI 組件 -->
</layout>
```

### 3. SingleLiveEvent 使用

```java
// 定義（com.cyberpower.edc.core.customobject.SingleLiveEvent）
public SingleLiveEvent<Void> executeClickEvent = new SingleLiveEvent<>();

// 觸發
executeClickEvent.call();

// 觀察
viewModel.executeClickEvent.observe(..., unused -> {
    // 執行邏輯
});
```

### 4. 自動滾動

```java
binding.tvLogMessages.post(() -> {
    ScrollView scrollView = (ScrollView) binding.tvLogMessages.getParent();
    scrollView.fullScroll(View.FOCUS_DOWN);
});
```

## 📝 使用方式

### 進入 API 測試

1. 啟動應用
2. 打開 Drawer（點擊 ☰）
3. 展開 PAX 或 CASTLES
4. 點擊「API測試」

### 執行測試

1. 在下拉選單選擇測試項目
2. 查看測試說明
3. 點擊「執行」按鈕
4. 觀察執行過程訊息

### 清除訊息

點擊「清除訊息」按鈕，訊息欄重置為「等待執行...」

### 返回

按返回鍵回到主畫面

## ⚠️ 已知問題

### 1. IDE 編譯警告

**問題**: SingleLiveEvent 類型無法解析  
**原因**: IDE 緩存問題  
**影響**: 僅 IDE 顯示，不影響實際編譯  
**解決**: 
```bash
# Android Studio
File → Invalidate Caches / Restart

# 或使用命令行
.\gradlew clean
.\gradlew assembleDebug
```

### 2. DataBinding BR 類

**問題**: BR.viewModel 無法識別  
**解決**: 
```bash
# Gradle 同步
File → Sync Project with Gradle Files

# 重新構建
Build → Rebuild Project
```

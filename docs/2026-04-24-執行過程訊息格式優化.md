# 執行過程訊息顯示格式優化

## 📅 更新日期
2026-04-24（最後更新：移除圖標符號）

## 🎯 修改原因

根據 PAX 機器的實際測試反饋，對執行過程訊息的顯示格式進行優化，使其更簡潔、更專注於核心信息。

---

## ✅ 完成的修改

### 1. 移除時間資訊 ⏰

**修改前**：
```
[12:34:56.789] ℹ️ [資訊] 設備型號: A920
[12:34:56.790] ℹ️ [資訊] 製造商: PAX
[12:34:56.791] ✅ [成功] 測試完成
```

**修改後**：
```
[資訊] 設備型號: A920
[資訊] 製造商: PAX
[成功] 測試完成
```

**原因**：
- ✅ 簡化顯示，讓訊息更易讀
- ✅ 測試執行通常很快，時間戳意義不大
- ✅ 減少每行長度，在小屏幕上更友好

### 2. 移除設備名稱 📱

**修改前** - `executeSystemInfoTest()` 顯示：
```
[資訊] 設備型號: A920
[資訊] 製造商: PAX
[資訊] Android 版本: 7.1.1
[資訊] SDK 版本: 25
[資訊] 設備名稱: A920    ← 移除此項
```

**修改後**：
```
[資訊] 設備型號: A920
[資訊] 製造商: PAX
[資訊] Android 版本: 7.1.1
[資訊] SDK 版本: 25
[資訊] 設備序號 (SN): A920ABC123456    ← 新增
```

**原因**：
- ✅ 設備名稱 (Build.DEVICE) 通常與型號重複
- ✅ 信息冗餘，沒有額外價值
- ✅ 騰出空間顯示更有用的信息（SN）

### 3. 增加設備序號 (SN) 🔑

**新增功能**：在系統資訊測試中顯示設備序號

**實現代碼**：
```java
// 獲取設備序號 (SN)
try {
    HardwareManager hwManager = HardwareManager.getInstance();
    if (hwManager != null && hwManager.getSys() != null) {
        String sn = hwManager.getSys().getSn();
        if (sn != null && !sn.isEmpty()) {
            addInfoMessage("設備序號 (SN): " + sn);
        } else {
            addWarningMessage("設備序號 (SN): 無法獲取");
        }
    } else {
        addWarningMessage("設備序號 (SN): 系統接口不可用");
    }
} catch (Exception e) {
    addWarningMessage("設備序號 (SN): 獲取失敗 - " + e.getMessage());
}
```

**顯示效果**：
```
✅ 成功獲取：
[資訊] 設備序號 (SN): A920ABC123456

⚠️ 無法獲取：
[警告] 設備序號 (SN): 無法獲取

⚠️ 接口不可用：
[警告] 設備序號 (SN): 系統接口不可用

❌ 獲取失敗：
[警告] 設備序號 (SN): 獲取失敗 - [錯誤信息]
```

**原因**：
- ✅ SN 是設備的唯一標識，非常重要
- ✅ 用於設備追蹤、維護、保修
- ✅ 在測試報告中需要記錄 SN
- ✅ 方便識別具體測試的設備

### 4. 移除圖標符號 🎯 (最新變更)

**修改前**：
```
ℹ️ [資訊] 設備型號: A920
✅ [成功] 測試完成
⚠️ [警告] 注意事項
❌ [錯誤] 測試失敗
```

**修改後**：
```
[資訊] 設備型號: A920
[成功] 測試完成
[警告] 注意事項
[錯誤] 測試失敗
```

**原因**：
- ✅ 某些設備可能無法正確顯示表情符號
- ✅ 顯示為 "|" 或其他亂碼字符
- ✅ 純文字更穩定、更兼容
- ✅ 保持簡潔清晰的顯示效果

## 🔧 技術實現

### 修改的代碼

#### 1. 移除時間格式化

**修改前**：
```java
private final SimpleDateFormat timeFormat = new SimpleDateFormat("HH:mm:ss.SSS", Locale.getDefault());

public void addLogMessage(String message) {
    String timestamp = timeFormat.format(new Date());
    String logEntry = "[" + timestamp + "] " + message + "\n";
    // ...
}
```

**修改後**：
```java
// 移除 timeFormat 字段

public void addLogMessage(String message) {
    // 不再添加時間戳，直接顯示消息
    String logEntry = message + "\n";
    // ...
}
```

#### 2. 清理導入

**修改前**：
```java
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.Locale;
```

**修改後**：
```java
// 移除上述導入，不再需要
```

#### 3. 更新系統資訊測試

**修改前**：
```java
public void executeSystemInfoTest() {
    addInfoMessage("========== 系統資訊測試 ==========");
    
    try {
        addInfoMessage("設備型號: " + android.os.Build.MODEL);
        addInfoMessage("製造商: " + android.os.Build.MANUFACTURER);
        addInfoMessage("Android 版本: " + android.os.Build.VERSION.RELEASE);
        addInfoMessage("SDK 版本: " + android.os.Build.VERSION.SDK_INT);

        addErrorMessage("系統資訊測試失敗: " + e.getMessage());
```

**修改後**：

## 📈 優點分析

### 1. 顯示更簡潔

**節省空間**：
- 每行減少約 14 個字符（時間戳）
- 每行減少約 2-3 個字符（圖標符號）
- 在小屏幕設備上更易讀
- 減少滾動需求

**視覺更清晰**：
```
最終版本（簡潔）：
[資訊] 設備型號: A920
[成功] 測試完成

VS

初始版本（冗長）：
[12:34:56.789] ℹ️ [資訊] 設備型號: A920
[12:34:56.790] ✅ [成功] 測試完成
```

### 2. 信息更有用

**移除冗餘**：
- 設備名稱通常與型號相同
- 避免重複信息

**添加關鍵**：
- SN 是設備唯一標識
- 對設備管理至關重要

### 3. 性能提升

**減少處理**：
```java
// 初始版本：每次都要格式化時間和添加圖標
String timestamp = timeFormat.format(new Date());
String logEntry = "[" + timestamp + "] ❌ [錯誤] " + message + "\n";

// 最終版本：直接使用消息
String logEntry = "[錯誤] " + message + "\n";
```

**效果**：
- ✅ 減少 Date 對象創建
- ✅ 減少字符串拼接操作
- ✅ 移除表情符號編碼
- ✅ 提升日誌添加速度

### 4. 兼容性更好

**避免顯示問題**：
- ✅ 某些 POS 設備可能不支持表情符號
- ✅ 表情符號可能顯示為 "|" 或亂碼
- ✅ 純文字在所有設備上都能正確顯示
- ✅ 字體支持更廣泛

## ⚠️ 注意事項

### 1. 日誌文件保留時間戳

雖然 UI 顯示不再有時間戳，但 **XLog 文件日誌仍然保留時間信息**：

```java
LogUtils.d(TAG, "添加日誌: " + message);  // ← XLog 自動添加時間戳
```

**XLog 文件示例**：
```
2026-04-24 12:34:56.789 D/ApiTestViewModel: 添加日誌: ℹ️ [資訊] 設備型號: A920
2026-04-24 12:34:56.790 D/ApiTestViewModel: 添加日誌: ✅ [成功] 測試完成
```

### 2. SN 獲取可能失敗

SN 獲取有完整的錯誤處理：

```java
✅ 正常：設備序號 (SN): A920ABC123456
⚠️ 為空：設備序號 (SN): 無法獲取
⚠️ null：設備序號 (SN): 系統接口不可用
❌ 異常：設備序號 (SN): 獲取失敗 - [具體錯誤]
```

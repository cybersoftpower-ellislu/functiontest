## 🔍 問題原因

### Android 應用啟動流程

```
用戶點擊圖示
    ↓
┌─────────────────────────────────────────┐
│ 1. 系統立即顯示預覽窗口（Preview Window） │
│    - 使用 Theme.Splash 的 windowBackground  │
│    - 原設定：@color/colorPrimary (藍色)    │
│    - 目的：立即給用戶視覺反饋               │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ 2. 應用程序進程啟動                        │
│    - Application.onCreate() 執行          │
│    - 初始化 Core、HardwareManager 等      │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ 3. SplashActivity 創建                    │
│    - onCreate() 執行                      │
│    - 權限檢查開始                          │
│    - 布局渲染                             │
└─────────────────────────────────────────┘
    ↓
用戶看到 SplashActivity 實際內容
```

**問題：** 步驟 1 的預覽窗口是純藍色，與步驟 3 的 SplashActivity 不一致，導致明顯的視覺跳變。

## 🔧 技術實現

### 1. 創建啟動畫面 Drawable

**檔案：** `app/src/main/res/drawable/splash_background.xml`

```xml
<layer-list xmlns:android="http://schemas.android.com/apk/res/android"
    android:opacity="opaque">
    
    <!-- 藍色背景（與 SplashActivity 一致）-->
    <item android:drawable="@color/colorPrimary" />
    
    <!-- 中央 Logo -->
    <item>
        <bitmap
            android:gravity="center"
            android:src="@mipmap/ic_launcher" />
    </item>
    
</layer-list>
```

**作用：**
- ✅ 背景色與 SplashActivity 相同（`colorPrimary`）
- ✅ 中央顯示應用 Logo
- ✅ 讓預覽窗口看起來像真正的啟動畫面

### 2. 更新主題配置

**檔案：** `app/src/main/res/values/themes.xml`

```xml
<!-- 修改前 -->
<style name="Theme.Splash" parent="...">
    <item name="android:windowBackground">@color/colorPrimary</item>
                                         ↑ 純藍色
</style>

<!-- 修改後 -->
<style name="Theme.Splash" parent="...">
    <item name="android:windowBackground">@drawable/splash_background</item>
                                         ↑ 自訂的啟動畫面 drawable
</style>
```

**效果：**
- ✅ 預覽窗口顯示帶 Logo 的啟動畫面
- ✅ 與 SplashActivity 視覺一致
- ✅ 消除藍色閃爍感

## 💡 為什麼這樣設計？

### Android 的設計哲學

1. **立即反饋**
   - 用戶點擊圖示後應立即看到反饋
   - 預覽窗口提供即時視覺確認

2. **感知性能**
   - 即使應用還在啟動，用戶也不會覺得"卡住"
   - 有內容的啟動畫面比空白/純色更專業

3. **品牌展示**
   - 啟動時是展示品牌的絕佳機會
   - Logo 和配色應保持一致

### Material Design 指南

> "Launch screens should be simple and unbranded, 
> but you can include your product logo."

**我們的實現：**
- ✅ 簡潔（藍色背景 + Logo）
- ✅ 包含品牌元素（Logo）
- ✅ 快速過渡（1.5 秒）
- ✅ 無干擾元素

## 🎨 進階自訂選項

### 選項 1：使用漸變背景

修改 `themes.xml`：
```xml
<item name="android:windowBackground">@drawable/splash_background_advanced</item>
```

效果：漸變藍色背景，更有質感。

### 選項 2：調整 Logo 大小

修改 `splash_background_advanced.xml`：
```xml
<item
    android:width="96dp"   <!-- 原 120dp -->
    android:height="96dp"
    android:gravity="center">
```

### 選項 3：更換背景顏色

創建新的顏色值：
```xml
<!-- colors.xml -->
<color name="splash_background">#1565C0</color>  <!-- 深藍色 -->
```

修改 `splash_background.xml`：
```xml
<item android:drawable="@color/splash_background" />
```

## ❓ 常見問題

### Q1: 修改後還是看到藍色閃爍？

**原因：**
- 應用未重新安裝（只是增量更新）
- 資源檔案未正確編譯

**解決：**
```
1. Build → Clean Project
2. 完全卸載應用
3. Build → Rebuild Project
4. 重新安裝
```

### Q2: Logo 顯示位置不對？

**原因：**
- Bitmap gravity 設定問題

**解決：**
```xml
<bitmap
    android:gravity="center"  <!-- 確認是 center -->
    android:src="@mipmap/ic_launcher" />
```

### Q3: Android 12+ 顯示不正常？

**原因：**
- Android 12 引入了新的 Splash Screen API

**解決：**
參考 `docs/冷啟動藍屏問題-解決方案.md` 中的 Android 12+ 方案。

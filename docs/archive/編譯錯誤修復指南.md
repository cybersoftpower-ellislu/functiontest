# 編譯錯誤修復指南

## 已修復的問題

### ✅ 1. API Level 不匹配
**錯誤訊息**: `Extending BaseActivity requires API level 24 (current min is 22)`

**原因**: Core 模組的 `BaseActivity` 要求 Android API 24 (Android 7.0)，但 app 模組的 `minSdk` 設定為 22

**修復**: 已更新 `app/build.gradle`
```gradle
minSdk 24  // 從 22 更新為 24
```

### ✅ 2. LogUtils 初始化順序
**原因**: LogUtils 依賴 XLog，而 XLog 在 `Core.init()` 中初始化

**修復**: 調整 `FunctionTestApplication` 中的初始化順序，確保在 `Core.init()` 之後才使用 LogUtils

## 🔧 解決步驟（必須在 Android Studio 中操作）

### 步驟 1: 開啟專案
```
File → Open → C:\DevAndroid\functiontest
等待 Android Studio 載入完成
```

### 步驟 2: 初始化 Git Submodule（如果需要）
如果 `core/` 資料夾是空的：
```bash
# 在 Android Studio 的 Terminal 中執行
git submodule update --init
```

### 步驟 3: Gradle Sync ⭐ 重要！
```
File → Sync Project with Gradle Files
或點擊工具列的 "Sync Now" 按鈕
```

等待 Sync 完成，這會：
- 下載相關依賴
- 設定專案結構
- 準備 DataBinding 生成環境

### 步驟 4: Clean Project
```
Build → Clean Project
```

等待清理完成（約 10-30 秒）

### 步驟 5: Rebuild Project ⭐ 關鍵步驟！
```
Build → Rebuild Project
```

這會：
- ✅ 生成 `ActivityMainBinding` 類別
- ✅ 生成 `BR` 類別（包含 `BR.viewModel`）
- ✅ 生成所有 R.id 資源
- ✅ 編譯所有 Java 代碼

等待編譯完成（約 1-3 分鐘）

### 步驟 6: 確認錯誤已解決
檢查 "Build" 視窗的輸出，應該顯示 "BUILD SUCCESSFUL"

如果仍有錯誤，繼續下一步驟。

### 步驟 7: Invalidate Caches（如果需要）
如果步驟 5 後仍有錯誤：
```
File → Invalidate Caches / Restart... → Invalidate and Restart
```

這會清除所有快取並重啟 IDE（需要 1-2 分鐘）

## ⚠️ 常見問題

### Q1: Gradle Sync 失敗
**可能原因**:
- 網路連線問題（需要下載依賴）
- Gradle 版本不兼容
- Core 子模組未正確載入

**解決方案**:
1. 檢查網路連線
2. 確認 `settings.gradle` 中包含 `include ':core'`
3. 執行 `git submodule update --init`

### Q2: Core 模組錯誤（找不到 BaseActivity 等）
**原因**: Core 子模組未載入

**解決方案**:
```bash
# 在專案根目錄執行
git submodule update --init
```

然後重新 Sync Project。

### Q3: DataBinding 類別仍然無法找到
**解決方案**:
1. 確認 `app/build.gradle` 中 `buildFeatures { dataBinding true }` 已添加
2. Clean → Rebuild Project
3. 如果還是失敗，Invalidate Caches / Restart

### Q4: API Level 錯誤
**錯誤**: `requires API level XX`

**確認**: `app/build.gradle` 中 `minSdk 24` 已設定

### Q5: 編譯成功但執行時崩潰
**可能原因**: 
- Core 未正確初始化
- 權限未授予

**檢查**:
1. 確認 `AndroidManifest.xml` 中 `android:name=".FunctionTestApplication"` 已設定
2. 在執行時授予相機、儲存等權限

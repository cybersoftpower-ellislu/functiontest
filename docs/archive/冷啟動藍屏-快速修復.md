# 冷啟動藍屏問題 - 快速修復指南

## ❓ 問題
**在顯示 SplashActivity 之前，出現一個藍色畫面**

## ✅ 原因
這是 Android 的**冷啟動預覽窗口**，使用了主題的 `windowBackground` 設定。

原始配置：
```xml
<!-- Theme.Splash 使用藍色背景 -->
<item name="android:windowBackground">@color/colorPrimary</item>
<!-- colorPrimary = #2196F3（藍色） -->
```

## 🔧 解決方案（已完成）

### 1️⃣ 創建了啟動畫面 drawable ✅

**檔案：** `app/src/main/res/drawable/splash_background.xml`

包含：
- 藍色背景（與 SplashActivity 一致）
- 中央 Logo

### 2️⃣ 更新了主題配置 ✅

**檔案：** `app/src/main/res/values/themes.xml`

```xml
<style name="Theme.Splash" parent="...">
    <!-- 從純色改為包含 Logo 的 drawable -->
    <item name="android:windowBackground">@drawable/splash_background</item>
</style>
```

## 🎨 進階自訂（選用）

### 使用漸變背景

如果想要更精美的啟動畫面：

**檔案：** `themes.xml`
```xml
<!-- 將此行 -->
<item name="android:windowBackground">@drawable/splash_background</item>

<!-- 改為 -->
<item name="android:windowBackground">@drawable/splash_background_advanced</item>
```

**效果：**
- 漸變藍色背景
- 固定大小的 Logo（120dp）

## 💡 原理說明

### Android 啟動流程

```
1. 用戶點擊圖示
   ↓
2. 系統顯示預覽窗口（Preview Window）
   - 使用 theme 的 windowBackground
   - 目的：立即給用戶視覺反饋
   ↓
3. 應用程序啟動
   ↓
4. SplashActivity 創建並顯示
```

### 關鍵點

- 預覽窗口是**系統行為**，無法跳過
- 但可以**自訂外觀**讓它看起來像真正的啟動畫面
- 關鍵是保持**視覺一致性**

## ❓ 常見問題

### Q: 還是看到藍色閃爍？

**A:** 
1. 確認已重新編譯專案
2. 完全刪除應用後重新安裝
3. 檢查 themes.xml 是否正確修改

### Q: Logo 太大/太小？

**A:** 
使用 `splash_background_advanced.xml` 並調整：
```xml
<item
    android:width="120dp"  <!-- 調整大小 -->
    android:height="120dp"
    android:gravity="center">
```

### Q: 想要白色背景？

**A:** 
修改 `splash_background.xml`：
```xml
<item android:drawable="@android:color/white" />
```
